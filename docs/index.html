Corticospinal Culture Simulation

Technologies: Python · Flask · NumPy · JavaScript · ECharts · Docker
Skills: Computational Neuroscience · Biophysical Modeling · Full-Stack Engineering · Web Visualization · RESTful API Design · Resource Optimization
Topics & Concepts: Hodgkin-Huxley Dynamics · AMPA/NMDA Synapses · Electrode-Stimulus Mapping · Buffered Data Rendering · Interactive Neural Visualization
Project Description:
Simulates a 2D culture of corticospinal neurons using Hodgkin-Huxley membrane dynamics with both fast (AMPA) and voltage-dependent (NMDA) synaptic models. The system predicts steady-state membrane behavior under customizable external current input. A RESTful Flask API runs the simulation and streams results to a JavaScript frontend. The UI renders a live network map showing neurons, synapses, and connections, alongside synchronized graphs for input stimuli and membrane potential traces. Buffered rendering smooths playback and ensures responsiveness under limited system resources. Users can insert electrodes via the UI to observe how stimulus location and strength affect network dynamics.

Neural Network From Scratch

Technologies:
C++ · STL(Standard Template Library) · Custom Header Architecture · Custom Logger
Skills:
Supervised Learning · Feedforward Network Implementation · Low-Level Memory Management · Modular System Design · Custom Tooling · CLI Interface Design · JNI Integration
Topics & Concepts: Weight Initialization · RMSProp Optimization · Gradient Descent · Leaky ReLU · Tanh Activation · Batch Training · Stochastic Training · Activation Function Integration
Project Description:
Built a modular feedforward neural network framework in C++ entirely from scratch with no external libraries. Supports batch-based supervised learning using both standard gradient descent and RMSProp optimization, configurable activation functions (Leaky ReLU, Tanh), and reference-based neuron connections. Includes a thread-safe, single-threaded custom logger for clean diagnostic output. Trained on the Iris dataset and visualized through a CLI interface that displays per-layer activations, error values, and expected outputs. Engineered for robustness, maintainability, and integration into larger AI pipelines.

Neural Connectivity & Synaptic Dynamics Toolkit

Technologies Used: Python · NumPy · Matplotlib · Shapely
Skills: Computational Neuroscience · Geometric Simulation · Scientific Programming · Software Engineering · Object-Oriented Design
Topics & Concepts: Tsodyks-Markram Synapse · Short-Term Plasticity · Probabilistic Synaptogenesis · Axon-Dendrite Overlap · Spatial Neural Modeling · Recursive Geometric Intersection
Project Description: Developed a modular simulation toolkit for generating spatially grounded networks of neurons and modeling synaptic plasticity. Synapses form probabilistically based on geometric overlap between axonal and dendritic growth regions, modeled as semicircular polygons and analyzed using Shapely. Connection probabilities follow an exponential function of intersection area, and synapse positions are computed from centroid integrals. The system also implements the full Tsodyks–Markram model for short-term synaptic dynamics, including continuous recovery/facilitation and spike-triggered updates. Recursive refinement and duplicate resolution allow scalable connectivity analysis. Visualizations show neuron positions, projection zones, and connectivity structure.

Java Maze Game

Technologies Used: Java · Swing · OOP
Skills: Procedural Generation · Game Architecture · Pathfinding Algorithms · Interactive UI Design · Software Engineering
Topics & Concepts: Maze Generation · Prim’s Algorithm · Breadth-First Search · Game State Management · Tile-Based Interaction · 2D Rendering
Project Description:
A 2D puzzle game featuring procedurally generated mazes with tile-based interactions and directional player movement. Mazes are generated using a variant of Prim’s Algorithm, with the player placed at the furthest tile from a randomly assigned goal via breadth-first search. Each maze includes special tiles—such as red (heal), purple (pause), and orange (damage over time)—that dynamically affect the player’s health, displayed as a numeric overlay. The game uses a modular entity system (Player, Goal, TrappedFloor, etc.) for extensible logic and visual behavior. Built with Java Swing for GUI layout, input handling, and animated tile rendering.

Texas Hold'em with Reinforcement Learning

Technologies: Python · TensorFlow · NumPy · Keyboard · Batch Scripting · PyInstaller
Skills: Reinforcement Learning · Evolutionary Optimization · Performance Optimization · Resource Management · Reward Engineering · Software Engineering · Object-Oriented Programming
Topics & Concepts: Dueling Double Deep Q-Networks (D3QN) · Epsilon-Greedy Exploration · Q-Value Decomposition · Simulation-Based Training · Experience Replay Buffers · Evolutionary Reward Function Optimization · Temporal Difference Learning · Poker State Encoding
Project Description:
Developed a reinforcement learning framework for Texas Hold’em Poker using a Dueling Double Deep Q-Network (D3QN) architecture. Each agent is assigned a unique reward weight vector applied to a shared reward function. After each generation of self-play, underperforming reward vectors are replaced by mutated versions of top performers. Agents are retrained with these new reward structures, evolving generalizable strategies across generations. The environment supports full betting logic, discrete action encoding, and CLI integration for training or manual testing. A batch script checks for Python installation and version compatibility at runtime—defaulting to a PyInstaller-built executable only if necessary—to ensure optimal performance across systems.
